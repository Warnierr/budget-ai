## Documentation Budget AI

Ce dossier centralise la documentation vivante du projet. Il complète les fichiers historiques (ROADMAP, ARCHITECTURE, etc.) à la racine.

### Structure

- `incidents/` : retours d'expérience, bugs majeurs, plans d'action.
- _à venir_ : spécifications fonctionnelles, design system, guides d'onboarding.

### Ajouter une nouvelle fiche

1. Créer un sous-dossier si nécessaire (ex : `incidents/`, `design/`…).
2. Nommer les fichiers avec la date ISO croissante `YYYY-MM-DD-titre.md`.
3. Documenter **symptômes → analyse → solution → actions futures**.
4. Lier la fiche depuis ce README pour retrouver l'information rapidement.
